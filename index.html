Section 01

Vector Physics

Unlike standard wheels that push only forward/backward, Mecanum wheels have rollers mounted at 45° to the wheel's plane — turning a simple rotation into a diagonal thrust vector.

🔑 By spinning four wheels in different combinations, these 45° vectors cancel in some directions and add together in others — producing movement in any direction.
↗️

45° Force Vectors

Each roller on a Mecanum wheel exerts force at a 45-degree angle relative to the robot's axis. This is the fundamental building block of omnidirectional movement.

➕

Vector Addition

When all four wheels spin forward, their 45° side-components cancel out (left vs right), and only forward thrust remains — that's how straight movement emerges from diagonal forces.

↔️

Strafing

For sideways movement, wheels spin in a pattern that cancels forward/backward components while the lateral (side) components all add together — pure sideways thrust with zero turning.

🔄

Rotation

For turning, left-side wheels push one direction while right-side wheels push the opposite — creating a rotating torque around the robot's center with zero linear motion.

Section 04

The Golden Rule: X-Pattern

This is the most common mistake for rookie teams. The wheels must be mounted so that the rollers form an "X" when viewed from above.

✓ CORRECT — "X" Pattern
✗ WRONG — "O" Pattern
✅

X Pattern (Correct)

Left wheels use "Left" rollers. Right wheels use "Right" rollers. When viewed from above, the top rollers of all 4 wheels point toward the center of the robot, forming an X.

❌

O Pattern (Wrong)

If all four wheels are the same type, the rollers form an "O". The robot can still rotate but will struggle to strafe — the lateral forces cancel instead of adding.

Section 05

Pros & Cons

Mecanum wheels are powerful but come with real trade-offs. Know them before your build season starts.

Advantages

  • ⚡Agility — Line up for a shot without backing up and turning. Save precious seconds during matches.
  • 🎯Precision — Combined with encoders, move exactly 12.5 inches sideways to the millimeter.
  • 📐Space efficiency — Navigate through gaps only 1 inch wider than your robot.

Trade-offs

  • ⚖️Weight — Mecanum wheels are significantly heavier than standard traction wheels.
  • 💪Traction — Rollers spin freely, reducing pushing power. A 6WD robot pinning you to the wall will likely win.
  • 💻Complex code — Requires four independent motor controllers and a specific math formula.
Section 06

Hardware Tips

🔧 Suspension / Flex

All 4 wheels must have equal pressure on the floor. A chassis warped just 1mm can cause one wheel to lift — your strafing will drift or arc unpredictably.

⚙️ Gear Ratios

Strafing fights internal friction. Recommended: 19.2:1 (~312 RPM) or 13.7:1 (~435 RPM) for a standard 4" Mecanum setup.

⚖️ Weight Distribution

Keep your Center of Gravity (CoG) centered over all 4 wheels. A rear-heavy robot will fishtail when strafing at speed.

Section 07

Programming Math

The standard Mecanum drive formula uses three input axes: y (forward/back), x (strafe), and rx (rotation). The x input is scaled by 1.1 to counteract imperfect strafing.

// ─── Mecanum Drive Formula ─────────────────────────────────────
// y  = left stick Y  (forward/backward)   range: [-1, 1]
// x  = left stick X  (strafe left/right)  range: [-1, 1]
// rx = right stick X (rotation)           range: [-1, 1]

double denominator = Math.max(Math.abs(y) + Math.abs(x) * 1.1 + Math.abs(rx), 1);

frontLeftPower  = (y + x * 1.1 + rx) / denominator;
backLeftPower   = (y - x * 1.1 + rx) / denominator;
frontRightPower = (y - x * 1.1 - rx) / denominator;
backRightPower  = (y + x * 1.1 - rx) / denominator;

// The denominator ensures no motor exceeds 100% power
// while maintaining the ratio between all four motors.
